1. Project Overview

The AI CV Analyzer is a Python-based application designed to analyze CVs (resumes) using Natural Language Processing (NLP) techniques. The system extracts text from PDF files, identifies key sections, detects skills and entities, and provides a comprehensive evaluation with actionable recommendations.

Purpose

  • Provide automated CV analysis for students and job seekers
  • Demonstrate NLP and AI-assisted analysis techniques
  • Serve as a foundation for future ATS-style tools
  • Educational project showcasing system design principles

Key Features

Accurate PDF text extraction
NLP-based section detection
Named Entity Recognition
Skill extraction & categorization
Explainable scoring system
Personalized recommendations

3. NLP Pipeline

The NLP pipeline processes extracted CV text through multiple stages to identify structure, entities, and content quality.

1

Text Preprocessing

Unicode normalization (NFC), character cleanup, line reconstruction, sentence segmentation.

2

Section Detection

Regex pattern matching for section headers combined with content-based heuristics for 9 CV sections.

3

Named Entity Recognition

spaCy's NER model identifies PERSON, ORG, GPE, DATE entities. Custom regex extracts emails and phone numbers.

4

Skill Extraction

Pattern matching against a comprehensive skills database covering programming languages, frameworks, databases, cloud/DevOps, data science, and soft skills.

5

Career Field Detection

Keyword frequency analysis across 13 career fields with skill correlation scoring.

6

Experience Level Classification

Keyword matching and year range extraction to classify: Fresher, Junior, Mid-Level, Senior.

4. Scoring System

The scoring system provides transparent, explainable results across five weighted categories.

Category Weight Factors
Experience Level 25% Detected level + work experience quality
Skills Relevance 25% Skill count + category diversity
CV Structure 20% Sections detected + section quality
Career Alignment 15% Field detection + skill-field match
Readability 15% Word count + bullet point usage

Explainable AI

Every score comes with detailed explanations. The system avoids "black box" scoring by providing clear reasoning for each rating, helping users understand exactly how to improve their CVs.

7. Ethical Considerations

Data Privacy

User data is stored locally and not shared with third parties. CVs are processed on-premises.

Bias Awareness

The scoring system uses objective, transparent criteria. However, users should be aware that any AI system may have inherent biases.

Transparency

All scoring factors and weights are documented. Users can understand exactly how their score was calculated.

Human Oversight

This tool is meant to assist, not replace, human judgment in CV evaluation. Final hiring decisions should always involve human review.

8. Limitations

  • English Only: Currently supports English language CVs only.
  • PDF Format: Only accepts PDF files. Other formats (DOCX, images) not supported.
  • Image-based PDFs: Scanned documents may not extract properly without OCR.
  • Complex Layouts: Highly stylized CVs with columns or graphics may have extraction issues.
  • Industry Coverage: Skill database may not cover all specialized industries.
